## 一、项目架构概览

这是一个基于 **RAG（检索增强生成）** 的智能知识库问答系统，采用前后端分离架构：

### 1.1 技术栈

| 层级 | 技术 |
|------|------|
| **后端** | Spring Boot 3.2.0 + Spring AI 1.1.2 |
| **前端** | Vue 3 + TypeScript + Element Plus |
| **数据库** | PostgreSQL + pgvector（向量存储）|
| **对象存储** | MinIO |
| **缓存** | Caffeine（本地）+ Redis（分布式）|
| **AI 模型** | 智谱 AI (GLM-4) / Ollama (本地模型) |

### 1.2 核心功能模块

1. **知识库管理** - 创建、编辑、删除知识库
2. **文档管理** - 多格式文档上传、智能分块、向量化
3. **智能问答** - RAG 问答、流式输出、多轮对话
4. **文档预览** - PDF/Word/Excel/PPT 在线预览

## 三、建议新增的功能模块

### 3.1 检索精度优化

**现状**: 单一向量检索

**建议**: 实现多路召回 + 重排序

| 优化项 | 说明 | 优先级 |
|--------|------|--------|
| BM25/全文检索 | 添加 Elasticsearch 作为补充召回路径 | 高 |
| Cross-Encoder 重排序 | 使用轻量级模型对召回结果精排 | 高 |
| 查询改写/扩展 | 使用 LLM 扩展用户查询的同义词和相关词 | 中 |

**实现思路**:
```
用户Query ───→ Query理解/改写 ───┐
                                 ↓
多路召回 ───→ 重排序 ───→ 上下文压缩 ───→ LLM
    ↑
┌───┴───────────┬──────────────┐
向量检索      关键词检索      知识图谱
```

### 3.2 用户与权限系统

**现状**: 无用户管理

**建议**: 添加完整的 RBAC 权限体系

| 功能模块 | 说明 |
|----------|------|
| 用户认证 | JWT Token 认证机制 |
| 角色管理 | 管理员/普通用户角色 |
| 权限控制 | 知识库级别的访问控制（公开/私有/指定用户）|
| 用户管理 | 用户注册、登录、密码重置 |

### 3.3 消息队列削峰

**现状**: 文档处理使用线程池 (`AsyncConfig.java`)

**建议**: 引入消息队列处理异步任务

| 优化项 | 说明 |
|--------|------|
| 消息队列选型 | RabbitMQ / RocketMQ / Kafka |
| 任务队列化 | 文档处理任务改为异步队列消费 |
| 任务重试 | 失败任务自动重试机制 |
| 死信队列 | 处理失败次数过多的任务 |
| 进度追踪 | 任务状态实时推送前端 |

### 3.4 监控与可观测性

**现状**: 仅有基础日志

**建议**: 添加完整监控体系

| 监控维度 | 技术选型 | 说明 |
|----------|----------|------|
| 指标采集 | Micrometer + Prometheus | QPS、延迟、错误率、Token 消耗 |
| 链路追踪 | Zipkin / Jaeger | 分布式调用链追踪 |
| 健康检查 | Spring Boot Actuator | 服务健康状态监控 |
| 告警机制 | Prometheus AlertManager | 异常自动告警 |
| 日志聚合 | ELK Stack / Loki | 集中化日志管理 |

### 3.5 前端功能扩展

**现状**: 基础聊天和文档管理

**建议**: 增强用户体验

| 功能 | 说明 |

| 对话导出 | 支持导出为 PDF/Markdown/Word |
| 统计报表 | 知识库使用统计、Token 消耗报表 |
