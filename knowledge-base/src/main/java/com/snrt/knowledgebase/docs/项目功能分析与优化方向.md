## 一、项目概述

### 1.1 项目简介

本项目是一个基于 **RAG（检索增强生成）** 技术的智能知识库问答系统，采用前后端分离架构，支持文档的智能解析、向量存储和语义检索问答。

### 1.2 技术架构

#### 后端技术栈

| Java | 17 | 编程语言 |
| Spring Boot | 3.2.0 | 应用框架 |
| Spring AI | 1.1.2 | AI 开发框架 |
| Spring Data JPA | - | 数据持久层 |

| pgvector | latest | 向量扩展插件 |

## 四、优化方向与建议

### 4.1 高优先级优化 (立即实施)

#### 4.1.1 完善多级缓存

**现状**: [RAGCacheManager.java](../service/RAGCacheManager.java) 已实现 L1/L2 缓存

**优化建议**:
- 添加语义缓存（L3）使用 SimHash/MinHash 快速去重
- 添加缓存预热机制
- 完善缓存命中率监控

**预期收益**: 热点 Query Token 消耗降低 80%+

#### 4.1.2 用户权限系统

**建议方案**:
- 使用 **Sa-Token + JWT** 实现认证授权
- 设计 RBAC 权限模型
- 知识库级别访问控制（公开/私有/指定用户）

**技术选型说明**:
- **Sa-Token**: 轻量级 Java 权限认证框架，功能丰富、使用简单
- **JWT**: 用于 Token 生成和校验，实现无状态认证

**数据库设计**:
```sql
-- 用户表
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    role VARCHAR(20) DEFAULT 'USER',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 知识库权限表
CREATE TABLE knowledge_base_permissions (
    id VARCHAR(36) PRIMARY KEY,
    knowledge_base_id VARCHAR(36) REFERENCES knowledge_base(id),
    user_id VARCHAR(36) REFERENCES users(id),
    permission VARCHAR(20) -- READ, WRITE, ADMIN
);
```

### 4.2 中优先级优化 (近期实施)

#### 4.2.1 检索精度优化

**现状**: 已实现多路召回 + RRF + 规则重排序

**进一步优化**:
- 接入 Elasticsearch 增强全文检索
- 实现真正的 Cross-Encoder 模型重排序
- 添加查询改写/扩展功能

```
用户Query ───→ Query理解/改写 ───┐
                                 ↓
多路召回 ───→ 重排序 ───→ 上下文压缩 ───→ LLM
    ↑
┌───┴───────────┬──────────────┐
向量检索      关键词检索      全文检索
```

#### 4.2.2 监控体系

**建议技术栈**:
- 指标采集: Micrometer + Prometheus
- 链路追踪: Zipkin / Jaeger
- 告警: Prometheus AlertManager
- 日志聚合: ELK Stack / Loki

**关键指标**:
- QPS、延迟、错误率
- Token 消耗统计
- 缓存命中率
- 文档处理成功率

#### 4.2.3 前端体验优化

**建议功能**:
- 对话导出 (PDF/Markdown/Word)
- 代码块一键复制
- 消息编辑和重新生成
- 响应式布局优化

### 4.3 低优先级优化 (后续规划)

#### 4.3.1 高级 RAG 特性

- **HyDE**: 用 LLM 生成假设答案再检索
