## 一、项目概述

### 1.1 技术栈分析

| 层级 | 技术选型 | 版本 |
|------|----------|------|
| 基础框架 | Spring Boot | 3.2.0 |
| Java版本 | Java | 17 |
| 数据访问 | Spring Data JPA | - |

### 1.2 当前架构问题总结

1. **职责边界模糊**：Service层混合了业务逻辑、数据转换、外部服务调用
2. **异常处理不规范**：使用RuntimeException直接抛出，缺乏结构化错误信息
3. **并发处理粗糙**：使用new Thread()创建异步任务，无法管理和监控
4. **缺乏缓存层**：热点数据频繁查询数据库
5. **安全隐患**：敏感配置明文存储，文件上传缺乏严格校验
6. **代码重复**：DTO转换、常量定义多处重复

## 二、架构设计层面优化

### 2.1 引入分层架构模式（DDD）

#### 当前问题
Service层职责过重，业务逻辑与数据访问混合，难以维护和测试。

#### 优化方案
引入领域驱动设计(DDD)分层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                        用户接口层                              │
│  Controller / DTO / 参数校验 / 接口文档                         │
├─────────────────────────────────────────────────────────────┤
│                        应用服务层                              │
│  ApplicationService / 用例编排 / 事务管理                       │
├─────────────────────────────────────────────────────────────┤
│                        领域层                                 │
│  Entity / ValueObject / DomainService / Repository接口         │
├─────────────────────────────────────────────────────────────┤
│                        基础设施层                              │
│  RepositoryImpl / 外部服务Client / 配置 / 工具类                │
└─────────────────────────────────────────────────────────────┘
```

#### 实施步骤
1. 创建`application`包，存放应用服务
2. 创建`domain`包，存放领域实体和服务
3. 将现有Service拆分为ApplicationService和DomainService
4. Repository保留接口，实现类移至infrastructure

#### 预期收益
- 职责分离，提高代码可维护性
- 便于单元测试，每层可独立测试
- 支持领域模型的演进

### 2.3 引入工厂模式创建ChatModel

#### 当前问题
ChatModelConfig使用switch-case选择模型，扩展性差。

#### 优化方案
使用工厂模式+依赖注入：

```java
// 1. 定义模型提供者接口
public interface ChatModelProvider {
    String getProviderName();
    ChatModel createModel();
}

// 2. 实现各个提供者
@Component
public class ZhipuAiModelProvider implements ChatModelProvider {
    @Override
    public String getProviderName() { return "zhipuai"; }
    // ...
}

@Component
public class OllamaModelProvider implements ChatModelProvider {
    @Override
    public String getProviderName() { return "ollama"; }
    // ...
}

// 3. 工厂类
@Component
public class ChatModelFactory {
    private final Map<String, ChatModelProvider> providers;
    
    public ChatModel getModel(String provider) {
        return Optional.ofNullable(providers.get(provider))
            .map(ChatModelProvider::createModel)
            .orElseThrow(() -> new UnsupportedModelProviderException(provider));
    }
}
```

#### 预期收益
- 新增模型提供商只需实现接口，无需修改工厂代码
- 符合开闭原则
- 便于单元测试，可Mock具体提供者

## 三、异常处理优化

### 3.1 自定义业务异常体系

#### 当前问题
使用RuntimeException抛出业务错误，缺乏结构化，前端难以处理。

#### 优化方案
创建完整的异常体系：

```java
// 1. 基础业务异常
public abstract class BusinessException extends RuntimeException {
    private final ErrorCode errorCode;
    private final Object[] args;
    
    public BusinessException(ErrorCode errorCode, Object... args) {
        super(MessageFormat.format(errorCode.getMessage(), args));
        this.errorCode = errorCode;
        this.args = args;
    }
}

// 2. 具体异常类型
public class ResourceNotFoundException extends BusinessException {
    public ResourceNotFoundException(String resourceType, String id) {
        super(ErrorCode.RESOURCE_NOT_FOUND, resourceType, id);
    }
}

public class ValidationException extends BusinessException {
    public ValidationException(String field, String message) {
        super(ErrorCode.VALIDATION_ERROR, field, message);
    }
}

public class ExternalServiceException extends BusinessException {
    public ExternalServiceException(String service, String message) {
        super(ErrorCode.EXTERNAL_SERVICE_ERROR, service, message);
    }
}
```

#### 预期收益
- 异常语义清晰，便于分类处理
- 支持错误码和参数化消息
- 便于统一日志记录和监控

### 3.3 全局异常处理增强

#### 当前问题
GlobalExceptionHandler过于简单，缺少详细错误信息。

#### 优化方案
扩展全局异常处理：

```java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public ApiResponse<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return ApiResponse.error(e.getErrorCode().getCode(), e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ApiResponse<Void> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
            .map(error -> error.getField() + ": " + error.getDefaultMessage())
            .collect(Collectors.joining(", "));
        return ApiResponse.error(ErrorCode.PARAM_ERROR.getCode(), message);
    }

    @ExceptionHandler(ConstraintViolationException.class)
    public ApiResponse<Void> handleConstraintViolation(ConstraintViolationException e) {
        String message = e.getConstraintViolations().stream()
            .map(ConstraintViolation::getMessage)
            .collect(Collectors.joining(", "));
        return ApiResponse.error(ErrorCode.PARAM_ERROR.getCode(), message);
    }

    @ExceptionHandler(DataAccessException.class)
    public ApiResponse<Void> handleDataAccessException(DataAccessException e) {
        log.error("数据库访问异常", e);
        return ApiResponse.error(ErrorCode.SYSTEM_ERROR.getCode(), "数据库操作失败");
    }

    @ExceptionHandler(Exception.class)
    public ApiResponse<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return ApiResponse.error(ErrorCode.SYSTEM_ERROR);
    }
}
```

#### 预期收益
- 细化的异常分类处理
- 统一的错误响应格式
- 详细的日志记录便于排查问题

## 四、并发与异步处理优化

### 4.1 使用线程池替代new Thread()

#### 当前问题
DocumentService中使用new Thread()创建异步任务，无法管理和监控。

```java
// 当前问题代码
private void processDocumentAsync(Document document) {
    new Thread(() -> {
        // 处理逻辑...
    }).start();
}
```

#### 优化方案
使用@Async注解+ThreadPoolTaskExecutor：

```java
// 1. 配置线程池
@Configuration
@EnableAsync
public class AsyncConfig {
    
    @Bean("documentProcessorExecutor")
    public Executor documentProcessorExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(4);
        executor.setMaxPoolSize(10);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("doc-processor-");
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        executor.initialize();
        return executor;
    }
}

// 2. 使用@Async注解
@Service
public class DocumentProcessingService {
    
    @Async("documentProcessorExecutor")
    public CompletableFuture<Void> processDocumentAsync(String documentId) {
        // 异步处理逻辑
        return CompletableFuture.completedFuture(null);
    }
}
```

#### 预期收益
- 线程池可配置、可监控
- 避免线程创建销毁的开销
- 支持任务队列和拒绝策略

### 4.2 添加异步任务状态追踪

#### 当前问题
文档处理异步任务无法追踪执行状态。

#### 优化方案
引入任务队列表：

```java
@Entity
@Table(name = "async_task")
public class AsyncTask {
    @Id
    private String id;
    
    @Enumerated(EnumType.STRING)
    private TaskType taskType;
    
    @Enumerated(EnumType.STRING)
    private TaskStatus status;
    
    private String targetId; // 关联的文档ID等
    
    @Column(columnDefinition = "TEXT")
    private String result; // 执行结果或错误信息
    
    private Integer progress; // 进度百分比
    
    private LocalDateTime createTime;
    private LocalDateTime startTime;
    private LocalDateTime completeTime;
}

// 任务追踪服务
@Service
public class AsyncTaskService {
    
    public String submitTask(TaskType type, String targetId) {
        AsyncTask task = new AsyncTask();
        task.setId(UUID.randomUUID().toString());
        task.setTaskType(type);
        task.setStatus(TaskStatus.PENDING);
        task.setTargetId(targetId);
        task.setCreateTime(LocalDateTime.now());
        taskRepository.save(task);
        
        // 提交异步执行
        processTaskAsync(task);
        
        return task.getId();
    }
    
    public TaskStatus getTaskStatus(String taskId) {
        return taskRepository.findById(taskId)
            .map(AsyncTask::getStatus)
            .orElseThrow(() -> new ResourceNotFoundException("任务", taskId));
    }
}
```

#### 预期收益
- 可查询任务执行状态
- 支持进度追踪
- 便于故障排查和重试

### 4.3 添加并发控制

#### 当前问题
向量存储操作缺乏并发控制，可能导致重复处理。

#### 优化方案
使用分布式锁：

```java
@Service
public class DocumentProcessingService {
    
    @Autowired private StringRedisTemplate redisTemplate;
    
    public void processAndIndexDocument(Path filePath, String documentId, String knowledgeBaseId) {
        String lockKey = "lock:document:" + documentId;
        Boolean locked = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, "1", Duration.ofMinutes(10));
        
        if (!Boolean.TRUE.equals(locked)) {
            log.warn("文档正在处理中，跳过重复处理: {}", documentId);
            return;
        }
        
        try {
            // 执行处理逻辑
        } finally {
            redisTemplate.delete(lockKey);
        }
    }
}
```

#### 预期收益
- 防止重复处理同一文档
- 支持分布式环境下的并发控制

// 1. 配置缓存
@Configuration
@EnableCaching
public class CacheConfig {
    
    @Bean
    public CacheManager cacheManager(RedisConnectionFactory factory) {
        RedisCacheConfiguration config = RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(10))
            .serializeKeysWith(RedisSerializationContext.SerializationPair
                .fromSerializer(new StringRedisSerializer()))
            .serializeValuesWith(RedisSerializationContext.SerializationPair
                .fromSerializer(new GenericJackson2JsonRedisSerializer()));
        
        return RedisCacheManager.builder(factory)
            .cacheDefaults(config)
            .build();
    }
}

### 6.2 添加接口限流保护

#### 当前问题
聊天接口和文件上传缺乏限流保护，存在被滥用的风险。

#### 优化方案

```java
// 1. 配置限流器
@Configuration
public class RateLimitConfig {
    
    @Bean
    public RateLimiter chatRateLimiter() {
        return RateLimiter.create(10.0); // 每秒10个请求
    }
    
    @Bean
    public RateLimiter uploadRateLimiter() {
        return RateLimiter.create(5.0); // 每秒5个上传请求
    }
}

// 2. 使用限流器
@RestController
@RequestMapping("/api/chat")
public class ChatController {
    
    @Autowired private RateLimiter chatRateLimiter;
    
    @PostMapping
    public ApiResponse<String> chat(@RequestBody ChatRequest request) {
        if (!chatRateLimiter.tryAcquire()) {
            return ApiResponse.error(429, "请求过于频繁，请稍后重试");
        }
        // 处理逻辑
    }
}
```

或使用Bucket4j实现更灵活的限流：

```java
@Service
public class RateLimitService {
    
    private final Map<String, Bucket> buckets = new ConcurrentHashMap<>();
    
    public boolean tryAcquire(String key, long capacity, Duration period) {
        Bucket bucket = buckets.computeIfAbsent(key, k -> 
            Bucket.builder()
                .addLimit(Bandwidth.classic(capacity, Refill.intervally(capacity, period)))
                .build()
        );
        return bucket.tryConsume(1);
    }
}
```

#### 预期收益
- 防止接口被恶意调用
- 保护系统资源

### 6.3 文件上传安全检查

#### 当前问题
文件上传缺乏类型和大小严格校验，存在安全风险。

#### 优化方案
添加文件安全检查：

```java
@Component
public class FileUploadValidator {
    
    // 允许的文件类型白名单
    private static final Set<String> ALLOWED_TYPES = Set.of(
        "txt", "md", "pdf", "doc", "docx", "json", "xml", "html"
    );
    
    // 最大文件大小 50MB
    private static final long MAX_FILE_SIZE = 50 * 1024 * 1024;
    
    // 文件类型魔数检查
    private static final Map<String, byte[]> FILE_MAGIC_NUMBERS = Map.of(
        "pdf", new byte[]{0x25, 0x50, 0x44, 0x46}, // %PDF
        "docx", new byte[]{0x50, 0x4B, 0x03, 0x04} // PK (zip格式)
    );
    
    public void validate(MultipartFile file) {
        // 1. 检查文件是否为空
        if (file == null || file.isEmpty()) {
            throw new ValidationException("file", "文件不能为空");
        }
        
        // 2. 检查文件大小
        if (file.getSize() > MAX_FILE_SIZE) {
            throw new ValidationException("file", "文件大小超过限制: " + MAX_FILE_SIZE);
        }
        
        // 3. 检查文件扩展名
        String extension = getFileExtension(file.getOriginalFilename());
        if (!ALLOWED_TYPES.contains(extension.toLowerCase())) {
            throw new ValidationException("file", "不支持的文件类型: " + extension);
        }
        
        // 4. 检查文件内容类型（魔数检查）
        if (FILE_MAGIC_NUMBERS.containsKey(extension)) {
            validateFileMagicNumber(file, extension);
        }
        
        // 5. 文件名安全检查（防止路径遍历）
        String filename = file.getOriginalFilename();
        if (filename.contains("..") || filename.contains("/") || filename.contains("\\")) {
            throw new ValidationException("file", "文件名包含非法字符");
        }
    }
    
    private void validateFileMagicNumber(MultipartFile file, String extension) {
        try (InputStream is = file.getInputStream()) {
            byte[] magic = FILE_MAGIC_NUMBERS.get(extension);
            byte[] fileHeader = new byte[magic.length];
            is.read(fileHeader);
            
            if (!Arrays.equals(magic, fileHeader)) {
                throw new ValidationException("file", "文件类型与扩展名不符");
            }
        } catch (IOException e) {
            throw new ValidationException("file", "文件读取失败");
        }
    }
}
```

#### 预期收益
- 防止恶意文件上传
- 防止路径遍历攻击
- 确保文件类型真实有效

@Component
public class TraceIdFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                     HttpServletResponse response, 
                                     FilterChain chain) throws ServletException, IOException {
        String traceId = request.getHeader("X-Trace-Id");
        if (traceId == null) {
            traceId = UUID.randomUUID().toString().replace("-", "");
        }
        
        MDC.put("traceId", traceId);

            chain.doFilter(request, response);
        } finally {
            MDC.clear();
        }
    }
}

### 8.1 DTO转换优化

#### 当前问题
多处使用手动setter进行DTO转换，代码冗余。

#### 优化方案
引入MapStruct：

```java
// 1. 定义Mapper
@Mapper(componentModel = "spring")
public interface KnowledgeBaseMapper {
    
    KnowledgeBaseMapper INSTANCE = Mappers.getMapper(KnowledgeBaseMapper.class);
    
    KnowledgeBaseDTO toDTO(KnowledgeBase entity);
    
    List<KnowledgeBaseDTO> toDTOList(List<KnowledgeBase> entities);
    
    @Mapping(target = "id", ignore = true)
    @Mapping(target = "createTime", ignore = true)
    @Mapping(target = "updateTime", ignore = true)
    @Mapping(target = "isDeleted", ignore = true)
    KnowledgeBase toEntity(KnowledgeBaseCreateRequest request);
}

// 2. 使用Mapper
@Service
@RequiredArgsConstructor
public class KnowledgeBaseService {
    
    private final KnowledgeBaseRepository repository;
    private final KnowledgeBaseMapper mapper;
    
    public KnowledgeBaseDTO getKnowledgeBase(String id) {
        KnowledgeBase kb = repository.findByIdAndIsDeletedFalse(id)
            .orElseThrow(() -> new ResourceNotFoundException("知识库", id));
        return mapper.toDTO(kb);
    }
}
```

#### 预期收益
- 减少样板代码
- 编译期类型安全
- 高性能（编译生成代码）

### 8.4 单元测试覆盖

class KnowledgeBaseServiceTest {

    @Mock private KnowledgeBaseMapper mapper;
    
    @InjectMocks private KnowledgeBaseService service;
    
    @Test
