# 智能知识库系统技术文档

## 项目概述

本项目是一个基于 RAG（检索增强生成）技术的智能知识库问答系统，采用前后端分离架构，集成了多种先进技术，实现了文档的智能解析、向量存储、语义检索和 AI 智能问答功能。

#### 代码示例
[VectorStoreRepository.java](file:///d:/yjr/code/knowledge-base/knowledge-base/src/main/java/com/snrt/knowledgebase/domain/document/repository/VectorStoreRepository.java) - 向量检索仓库
```java
@Repository
public class VectorStoreRepository {
    
    @PersistenceContext
    private EntityManager entityManager;

    /**
     * BM25全文检索
     */
    public List<VectorDocument> searchByBM25AndKnowledgeBase(
            String tsQuery, String knowledgeBaseId, int topK, int normalizationMode) {
        String sql = """
            SELECT d.id AS document_id, d.name AS document_name, 
                   d.knowledge_base_id, v.content, 
                   ts_rank_cd(v.ts_vector, to_tsquery(:query), :norm) as rank
            FROM vector_store v
            JOIN document d ON v.metadata->>'document_id' = d.id
            WHERE d.knowledge_base_id = :kbId
              AND v.metadata->>'is_chunk' = 'false'
            ORDER BY rank DESC
            LIMIT :limit
            """;
        
        return entityManager.createNativeQuery(sql, VectorDocument.class)
                .setParameter("query", tsQuery)
                .setParameter("kbId", knowledgeBaseId)
                .setParameter("limit", topK)
                .setParameter("norm", normalizationMode)
                .getResultList();
    }
}
```

#### 解决的问题
- **向量存储**：pgvector 提供高效的向量存储和索引
- **混合检索**：结合向量检索和全文检索（BM25）
- **HNSW索引**：提供高效的近似最近邻搜索

        template.setConnectionFactory(connectionFactory);

@Configuration
public class RabbitMQConfig {

    public static final String DOCUMENT_PROCESS_QUEUE = "document.process.queue";
    public static final String DOCUMENT_PROCESS_EXCHANGE = "document.process.exchange";
    public static final String DOCUMENT_PROCESS_ROUTING_KEY = "document.process";

    public static final String DOCUMENT_PROCESS_DLQ = "document.process.dlq";
    public static final String DOCUMENT_PROCESS_DLX = "document.process.dlx";

    @Bean
    public Queue documentProcessQueue() {
        return QueueBuilder.durable(DOCUMENT_PROCESS_QUEUE)
                .withArgument("x-dead-letter-exchange", DOCUMENT_PROCESS_DLX)
                .withArgument("x-dead-letter-routing-key", DOCUMENT_PROCESS_DLQ_ROUTING_KEY)
                .withArgument("x-message-ttl", 300000)
                .build();
    }

    @Bean
    public Queue documentProcessDLQ() {
        return QueueBuilder.durable(DOCUMENT_PROCESS_DLQ).build();
    }

    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory, 
                                         MessageConverter messageConverter) {
        RabbitTemplate template = new RabbitTemplate(connectionFactory);
        template.setMessageConverter(messageConverter);
        template.setConfirmCallback((correlationData, ack, cause) -> {
            if (!ack) {
                log.error("消息发送失败：{}", cause);
            }
        });
        return template;
    }
}

[DocumentProcessConsumer.java](file:///d:/yjr/code/knowledge-base/knowledge-base/src/main/java/com/snrt/knowledgebase/infrastructure/messaging/DocumentProcessConsumer.java) - 文档处理消费者
```java
@Component
public class DocumentProcessConsumer {

    @RabbitListener(queues = RabbitMQConfig.DOCUMENT_PROCESS_QUEUE, concurrency = "3-10")
    @Transactional
    public void handleDocumentProcess(DocumentProcessMessage message) {
        String documentId = message.getDocumentId();
        log.info("[文档处理] 收到消息: documentId={}", documentId);

        try {
            Document document = documentRepository.findByIdAndIsDeletedFalseWithKnowledgeBase(documentId)
                    .orElse(null);
            
            Path filePath = storageService.ensureLocalFileForProcessing(document);
            DocumentProcessor processor = processorFactory.getProcessor(document.getType());
            processor.processDocument(filePath, document.getId(), document.getName(),
                    knowledgeBaseId, knowledgeBaseName);

            document.setStatus(Document.DocumentStatus.COMPLETED);
            documentRepository.save(document);

        } catch (Exception e) {
            log.error("[文档处理] 处理失败: documentId={}", documentId, e);
            handleProcessFailure(message, e);
        }
    }
}
```

#### 解决的问题
- **异步处理**：文档上传后异步处理，不阻塞用户请求
- **削峰填谷**：支持高并发文档上传场景
- **重试机制**：死信队列支持消息重试
- **并发消费**：支持多消费者并行处理

### 14. 多路召回检索

#### 引入原因
结合向量检索和全文检索的优势，提升检索质量。

#### 代码示例
[MultiRetriever.java](file:///d:/yjr/code/knowledge-base/knowledge-base/src/main/java/com/snrt/knowledgebase/infrastructure/retrieval/MultiRetriever.java) - 多路召回
```java
@Component
public class MultiRetriever {

    public List<Document> retrieve(String query, String knowledgeBaseId, int topK) {
        CompletableFuture<List<Document>> vectorFuture = 
            submitVectorRetrievalTask(query, knowledgeBaseId, topK * 2, traceId);
        CompletableFuture<List<Document>> fullTextFuture = 
            submitFullTextRetrievalTask(query, knowledgeBaseId, topK, traceId);

        CompletableFuture.allOf(vectorFuture, fullTextFuture).join();

        List<Document> vectorResults = vectorFuture.get();
        List<Document> fullTextResults = fullTextFuture.get();

        List<Document> fusedResults = performRrfFusion(vectorResults, fullTextResults, topK, traceId);
        return fusedResults;
    }

    private List<Document> performRrfFusion(List<Document> vectorResults, 
                                           List<Document> fullTextResults, 
                                           int topK, String traceId) {
        return reciprocalRankFusion(vectorResults, fullTextResults, topK);
    }

    private List<Document> reciprocalRankFusion(List<Document> vectorResults,
                                               List<Document> keywordResults,
                                               int topK) {
        final int K = 60;
        Map<String, Document> docMap = new HashMap<>();
        Map<String, Double> scoreMap = new HashMap<>();

        processRetrievalResults(vectorResults, K, docMap, scoreMap);
        processRetrievalResults(keywordResults, K, docMap, scoreMap);

        return scoreMap.entrySet().stream()
                .sorted(Map.Entry.<String, Double>comparingByValue().reversed())
                .limit(topK)
                .map(entry -> {
                    Document doc = docMap.get(entry.getKey());
                    doc.getMetadata().put("rrf_score", entry.getValue());
                    return doc;
                })
                .collect(Collectors.toList());
    }
}
```

#### 解决的问题
- **并行检索**：向量检索和全文检索并行执行
- **结果融合**：RRF 算法融合多路结果
- **性能优化**：异步执行提升检索效率

### 17. Cross-Encoder 重排序

#### 引入原因
对召回结果进行精排，提升检索质量。

#### 代码示例
[CrossEncoderReranker.java](file:///d:/yjr/code/knowledge-base/knowledge-base/src/main/java/com/snrt/knowledgebase/infrastructure/retrieval/CrossEncoderReranker.java) - 重排序器
```java
@Component
public class CrossEncoderReranker {

    public List<Document> rerankByRules(String query, List<Document> documents, int topK) {
        String queryLower = query.toLowerCase();
        List<String> queryTerms = Arrays.asList(queryLower.split("\\s+"));

        List<Document> scoredDocs = documents.stream()
                .map(doc -> {
                    double score = calculateRelevanceScore(doc, queryTerms, queryLower);
                    doc.getMetadata().put("rule_score", score);
                    return doc;
                })
                .collect(Collectors.toList());

        List<Document> rerankedDocs = scoredDocs.stream()
                .sorted((d1, d2) -> {
                    Double s1 = (Double) d1.getMetadata().get("rule_score");
                    Double s2 = (Double) d2.getMetadata().get("rule_score");
                    return Double.compare(s2, s1);
                })
                .limit(topK)
                .collect(Collectors.toList());

        return rerankedDocs;
    }

    private double calculateRelevanceScore(Document doc, List<String> queryTerms, String queryLower) {
        String text = doc.getText().toLowerCase();
        double score = 0.0;

        // 1. 完整查询匹配
        if (text.contains(queryLower)) {
            score += 10.0;
        }

        // 2. 关键词匹配
        for (String term : queryTerms) {
            int count = countOccurrences(text, term);
            score += count * 2.0;

            // 标题匹配
            Object title = doc.getMetadata().get("document_name");
            if (title != null && title.toString().toLowerCase().contains(term)) {
                score += 5.0;
            }
        }

        // 3. 向量检索分数
        Object vectorScore = doc.getMetadata().get("rrf_score");
        if (vectorScore instanceof Number) {
            score += ((Number) vectorScore).doubleValue() * 5.0;
        }

        // 4. 知识图谱分数
        Object kgScore = doc.getMetadata().get("kg_score");
        if (kgScore instanceof Number) {
            score += ((Number) kgScore).doubleValue() * 8.0;
        }

        return score;
    }
}
```

#### 解决的问题
- **规则重排**：基于关键词匹配的轻量级重排
- **多维度评分**：综合考虑多种评分维度
- **性能优化**：避免频繁调用 LLM

## 总结

本项目采用了多种先进的技术栈，实现了高性能、高可用、易扩展的智能知识库系统。核心技术包括：

1. **Spring Boot 3.2.0**：现代化的应用框架
2. **Spring AI 1.1.2**：AI 开发框架
3. **PostgreSQL + pgvector**：向量数据库
4. **Redis + Caffeine**：多级缓存
5. **RabbitMQ**：异步消息处理
6. **Neo4j**：知识图谱
7. **MinIO**：对象存储
8. **多路召回 + HyDE + 查询改写**：高级 RAG 技术

项目具有以下特点：

- ✅ **高性能**：多级缓存、异步处理、并行检索
- ✅ **高可用**：Docker 容器化部署、故障自动恢复
- ✅ **易扩展**：插件化架构、配置化管理
- ✅ **智能化**：LLM 驱动的智能检索和问答
- ✅ **易维护**：统一异常处理、统一响应格式、完整日志
